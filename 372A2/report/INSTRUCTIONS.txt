Assignment.2 (Group) – Reliable Data Transfer over UDP
CP372 – Computer Network, Spring 2026


1. Overview
In this assignment, you will implement and compare two fundamental reliable data transfer protocols:
	•	Stop-and-Wait 
	•	Go-Back-N (GBN)
Both protocols will be implemented on top of UDP sockets in Python.
Since UDP does not provide reliability, your implementation must add mechanisms such as:
	•	Sequence numbers
	•	Acknowledgments (ACKs)
	•	Timeouts
	•	Retransmissions
The objective is to understand how transport protocols achieve reliability despite packet loss.

2. Learning Objectives
After completing this assignment, students should be able to:
	•	Explain the limitations of UDP.
	•	Implement reliable data transfer mechanisms.
	•	Implement Stop-and-Wait.
	•	Implement Go-Back-N.
	•	Handle packet loss through retransmissions.
	•	Compare protocol performance under different network conditions.
	•	Analyze throughput and efficiency of pipelined transmission.

Part A: Stop-and-Wait Protocol (35 Marks)
Requirements
Implement Stop-and-Wait.
Sender
The sender must:
	•	Read a text file,  or generate some random/constant content by the program.
	•	notify the server a file is to be transferred over
	•	Divide the file into packets.
	•	Send one packet at a time.
	•	Start a timer after transmission.
	•	Wait for ACK.
	•	Retransmit upon timeout.
	•	Continue until all packets are acknowledged.
	•	Send a packet to notify the completion of file transfer 

Receiver
The receiver must:
	•	Receive packets.
	•	Check sequence number, Send ACKs for in order packets
	•	Check packet TYPE, create a file if the TYPE = “upload a file”, save file content if TYPE == DATA; close the file if the TYPE == “completion of file transfer”
	•	Ignore duplicate packets or out order packets.
